## 高影响变化详细分析

### 1. Dream 记忆整合系统（ch24 重大更新）

**新增信号**：
- `tengu_dream_invoked` — 手动 `/dream` 触发
- `tengu_auto_dream_skipped` — 自动 dream 跳过（条件不满足：会话数不够等）
- `tengu_auto_dream_fired` / `_completed` / `_failed` / `_toggled` — 自动 dream 生命周期
- `tengu_kairos_dream` — 后台定时 dream（cron 调度）

**从 bundle 提取的 dream 提示词**：
1. "You are performing a dream — a pruning pass over your memory files"（修剪）
2. "You are performing a dream — a reflective pass over your memory files. Synthesize what..."（反思）
3. "Do not promote personal memories into `team/` during a dream"

**v2.1.88 架构参考**：
- `restored-src/src/services/memory/` — v2.1.88 记忆系统只有基本的 extract_memories
- Dream 是 v2.1.100 全新引入的子系统，没有 v2.1.88 对应物

**对书的影响**：ch24 需要新增完整的"Dream 记忆整合"小节（约 1500-2000 字），包含三种触发模式、两种 dream 类型、锁机制、autoDreamEnabled 设置。

---

### 2. Auto Mode 权限（ch16 重大更新）

**信号证据**：
- sdk-tools.d.ts 新增 `"auto"` 在 permission mode 枚举中
- `INTERNAL_PERMISSION_MODES` 包含 auto
- Auto mode 的行为：跳过权限检查（`if(q==="auto"||q==="bypassPermissions") return!0`）
- 使用 TRANSCRIPT_CLASSIFIER 对工具调用进行安全分类

**v2.1.88 架构参考**：
- `restored-src/src/utils/permissions/` — 5 种权限模式（default, acceptEdits, bypassPermissions, dontAsk, plan）
- v2.1.100 新增第 6 种：auto

**对书的影响**：ch16 需要将权限模式表从 5 种扩展到 6 种，新增 Auto Mode 说明小节。

---

### 3. Advisor 工具（ch21 潜在新章节）

**信号证据**：
- `tengu_advisor_command` / `_dialog_shown` — 事件追踪
- `advisorModel` 配置字段 — 可配置 advisor 模型
- 完整的 advisor 提示词（从 bundle 提取）：
  - "advisor tool backed by a stronger reviewer model"
  - "Call advisor BEFORE substantive work"
  - "Call advisor at least once before committing to an approach"
  - "If advisor points another: don't silently switch. Surface the conflict"

**对书的影响**：这是一个跨越 ch06b（API）和 ch21（Effort/Thinking）的新特性。建议在 ch21 新增 Advisor 小节。

---

### 4. Bedrock/Vertex 完整设置向导（ch06b + 附录 G）

**信号证据**：18 个新事件覆盖完整的设置和升级生命周期：
- 设置向导：`setup_started` → `setup_complete` / `setup_cancelled`
- 模型探测：`probe_result` — 验证模型 ID 可用性
- 默认模型检查：`default_check` → `default_fallback`
- 模型升级：`upgrade_check` → `upgrade_accepted` / `upgrade_declined` → `upgrade_relaunch` / `upgrade_save_failed`

**v2.1.88 架构参考**：
- `restored-src/src/services/api/bedrock.ts` — v2.1.88 只有基本 Bedrock 支持
- v2.1.100 引入完整的交互式设置向导和自动升级检测

---

### 5. Tool Result Dedup（ch10 重要更新）

**信号证据**：
```
"refer to that output" — 用短 ID 引用之前的工具结果
tengu_tool_result_dedup: hit (命中去重) / miss (未命中)
originalBytes / savedBytes — 追踪节省量
```

**对书的影响**：ch10（压缩后的文件状态保留）需要新增"工具结果去重"小节，这是一个重要的上下文优化机制。

---

### 6. Bridge REPL 重构（ch22 重大更新）

16 个 `tengu_bridge_repl_*` 事件在 v2.1.100 中完全移除，同时 `CLAUDE_CODE_REPL` 环境变量移除。这暗示 IDE 桥接通信机制经历了重大重构。

**v2.1.88 架构参考**：
- `restored-src/src/services/bridge/` — REPL 模式基于轮询的桥接
- v2.1.100 可能转向事件驱动或 WebSocket 方案
