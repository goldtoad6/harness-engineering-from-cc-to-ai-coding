## 3. sdk-tools.d.ts 变化

### 3.1 新增 `toolStats` 字段

```typescript
toolStats?: {
  readCount: number;
  searchCount: number;
  bashCount: number;
  editFileCount: number;
  linesAdded: number;
  linesRemoved: number;
  otherToolCount: number;
};
```

会话级工具使用统计，可用于成本估算、行为分析。

### 3.2 新增 `"auto"` 权限模式

```diff
- mode?: "acceptEdits" | "bypassPermissions" | "default" | "dontAsk" | "plan";
+ mode?: "acceptEdits" | "auto" | "bypassPermissions" | "default" | "dontAsk" | "plan";
```

Auto 模式：使用 TRANSCRIPT_CLASSIFIER 自动审批工具执行，无需用户确认。

### 3.3 新增 `staleReadFileStateHint` 字段

```typescript
/**
 * Model-facing note listing readFileState entries whose mtime bumped
 * during this command (set when WRITE_COMMAND_MARKERS matches)
 */
staleReadFileStateHint?: string;
```

工具执行期间检测文件 mtime 变化，向模型提示陈旧读取状态。

### 3.4 `originalFile` 放宽为可空

```diff
- originalFile: string;
+ originalFile: string | null;
```

## 8. 关键新功能详细分析

### 8.1 Advisor 工具（服务端 advisor）

从 bundle 中提取的 advisor 工具提示词：
- 使用更强的 reviewer 模型（如 Opus 4.6 审阅 Sonnet 4.6 的工作）
- 无参数调用 — 自动转发整个对话历史
- 规则："在实质性工作之前调用 advisor"、"提交前至少调用一次"
- 如果 advisor 结果与已有数据冲突，要求"表面化冲突，而非静默切换"

### 8.2 Dream 系统（记忆整合）

v2.1.100 引入完整的 "dream" 记忆整合系统：
- 手动触发（`/dream`）+ 自动触发（`auto_dream`）+ 后台定时（`kairos_dream`）
- 两种 dream 类型：修剪（pruning pass）和反思（reflective pass）
- 支持团队记忆（`team/` 目录）与个人记忆的分离
- 锁机制防止并发 dream 冲突
- 可通过设置开关（`autoDreamEnabled`）

### 8.3 Tool Result Dedup（工具结果去重）

新增工具结果去重机制：
- 检测重复的工具调用结果
- 用短引用 ID（如 `r1`, `r2`）替换重复内容
- 追踪节省的字节数（`savedBytes`）
- 有效减少上下文消耗

### 8.4 Autofix PR（自动修复 PR）

全新的远程代理任务类型：
- 生成远程 Claude Code 会话监控和自动修复 PR
- 支持分支重用（`reuseOutcomeBranch`）
- 与 ultraplan、ultrareview 并列为远程代理任务类型

### 8.5 Perforce 版本控制支持

新增 Perforce 工作区检测和支持：
- 通过 `.p4config` 文件或 `P4PORT` 环境变量检测
- 添加 Perforce 模式提示词（只读文件需要 `p4 edit` 打开）
- 与 git、mercurial、svn、jujutsu、sapling 并列
